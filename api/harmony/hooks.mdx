# Hooks

Harmony provides custom React hooks for common UI patterns.

## useIsMobile

A hook that detects whether the viewport is below the mobile breakpoint (768px).

```tsx
import { useIsMobile } from '@gigadrive/harmony';

function MyComponent() {
  const isMobile = useIsMobile();

  return (
    <div>
      {isMobile ? (
        <MobileNavigation />
      ) : (
        <DesktopNavigation />
      )}
    </div>
  );
}
```

### Return Value

<ResponseField name="isMobile" type="boolean">
  `true` when viewport width is less than 768px, `false` otherwise. Initially `undefined` during SSR, then updates on mount.
</ResponseField>

### Implementation Details

- Uses `window.matchMedia` for efficient resize detection
- Updates only when crossing the 768px breakpoint (not on every resize)
- SSR-safe: returns `false` on initial render, updates in `useEffect`
- Automatically cleans up event listeners on unmount

### Example: Responsive Component

```tsx
import { useIsMobile } from '@gigadrive/harmony';
import { Dialog, Drawer } from '@gigadrive/harmony';

function ResponsiveModal({ children }) {
  const isMobile = useIsMobile();
  
  // Use Drawer on mobile, Dialog on desktop
  const Component = isMobile ? Drawer : Dialog;
  
  return (
    <Component>
      {children}
    </Component>
  );
}
```

### Breakpoint Constant

The mobile breakpoint is defined as:

```tsx
const MOBILE_BREAKPOINT = 768; // pixels
```

This matches Tailwind CSS's default `md` breakpoint.

## Performance Considerations

- Uses `matchMedia` API for efficient resize detection (no resize event throttling needed)
- Only re-renders when crossing the breakpoint, not on every pixel change
- Minimal memory footprint (single event listener)
